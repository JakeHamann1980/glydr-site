Getting Started
New to the brand kit? This guide explains what everything is, how to use it, and how to get assets ready for production.
What Is a Brand Kit?
A brand kit is the single source of truth for how GLYDR looks, sounds, and feels across every touchpoint — website, ads, social media, packaging, and more. It ensures that no matter who creates content for GLYDR, the result is consistent and professional.

Think of it as a recipe book: the ingredients (colors, fonts, logos) and instructions (rules, spacing, tone) that guarantee every dish tastes like GLYDR.
●
Why It Matters
Consistent branding builds trust and recognition. When everything matches, customers remember you.
●
Who Uses It
Anyone creating content for GLYDR — designers, marketers, contractors, agencies, or even Claude.
●
Keep It Updated
As the brand evolves, this kit should too. It's a living document, not a one-time deliverable.
What's in Each Section
Logo System
The GLYDR wordmark and icon in all approved variations (dark, light, cyan backgrounds). Download SVGs here. Never recreate the logo in text.
Color
Every approved color with hex codes, CSS variable names, and usage rules. Copy hex codes directly from the swatches. Cyan = primary, Volt = secondary, Ember = urgency only.
Typography
The three font families (Barlow Condensed, Barlow, Share Tech Mono) with weight/size rules. Headings are always Condensed uppercase. Body is regular Barlow. Mono is for data and labels.
Buttons & Components
Ready-made UI patterns — buttons, badges, cards, inputs. Each has size variants and hover states. Use these as templates when building new pages or ads.
Banner Ads
30+ animated banner templates in every standard ad size. Hover to preview animations. Use approved copy and brand colors — don't invent new claims.
Voice & Tone
How GLYDR sounds in writing. Approved headlines, do/don't examples, and the four voice attributes: Confident, Technical, Inclusive, Playful.
How to Review & Grab Assets
Follow these steps to find and use brand assets:
1
Navigate
Use the sidebar to jump to the section you need. Each section has labeled subsections.
2
Inspect
Right-click any element and "Inspect" to see exact CSS values. Colors show hex codes below each swatch.
3
Copy
Hex codes, font names, and CSS variable names are visible on every component. Copy them directly into your design tool or code.
4
Download
SVG logos are in assets/svg/. Product photos in assets/img/. Video in assets/video/.
Staging Ads for Production
Step 1: Choose a Template
Go to the Banner Ads section and pick a template that matches your campaign goal (awareness, urgency, community, etc.) and required ad size.
Step 2: Customize Copy
Swap in approved taglines from the Voice & Tone section or from features.json taglines. Never invent claims that aren't approved.
Step 3: Export
For static ads: screenshot at the banner's native dimensions. For animated: use a screen recorder or rebuild in your ad platform using the exact brand tokens.
Step 4: Review Checklist
Run through the checklist below before shipping anything.
Don't
Use colors not in the palette (no purple, no rainbow RGB).
Don't
Recreate the logo with text — always use SVG files.
Don't
Invent product claims. Only use copy from this kit or glydr.gg.
Don't
Use emojis in professional ads or UI.
Don't
Advertise features not yet activated in features.json (e.g., Bluetooth).
Pre-Ship Checklist
Logo: Using official SVG files (not text recreation)
Colors: All colors match hex values from the Color section
Typography: Barlow Condensed for headings, Barlow for body, Share Tech Mono for labels
Copy: All text is approved copy from Voice & Tone or glydr.gg
Features: No blocked features advertised (check features.json)
Dimensions: Ad matches required platform size (check banner label)
CTA: Clear call-to-action button with correct price ($299)
Links: All URLs point to glydr.gg or approved social links
Feature Scheduling (features.json)
GLYDR uses a features.json file to control which product features can be advertised. This prevents accidentally promoting capabilities that aren't ready or licensed yet.
●
Active
Feature is live and can be advertised everywhere.
●
Scheduled
Has a future date. Content will auto-show on that date. Plan campaigns around it.
●
Planned
Coming eventually but no date set. Don't advertise yet.
●
Blocked
Cannot activate (legal, budget, or tech). Example: Bluetooth licensing.
// To schedule a feature: // 1. Open features.json // 2. Find the feature // 3. Set activateDate and status "bluetooth": { "activateDate": "2025-09-01", "status": "scheduled" } // Site content with // data-feature="bluetooth" // will auto-show on Sep 1
Managing Taglines
All approved taglines are stored in features.json under the taglines key. To add a new tagline:
// In features.json → taglines → hero: "hero": { "primary": "Unlock Your Feet.", "alternatives": [ "Your Hands Are Maxed Out.", "Engage the Agility of Your Feet.", "Add your new tagline here." // ← just add to the array ] }
Categories available: hero, sub, tagline, product, cta_primary, cta_secondary, ticker. The primary is used by default; alternatives are available for A/B testing or campaign variations.

Usage Rules
✓ Do
  • Use Barlow Condensed 900 only
  • Maintain 0.5× height clearance on all sides
  • Use cyan Y-accent on dark / surface BGs
  • Ghost/outline variant for large display only
✗ Don't
  • Never stretch, skew, or rotate the mark
  • Never use cyan accent on light backgrounds
  • Never add drop shadows or filters
  • Never place on busy photos without a scrim
