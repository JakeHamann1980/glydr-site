SaaS Evolution Path
Starting as an internal marketing ops tool for consulting engagements. Architecture is designed to support multi-tenant SaaS if the tool proves valuable enough to productize.
PHASE 1 — NOW
Internal Tool
Single-tenant admin panel. Password auth. Used by GLYDR team only. Brand kit + tagline management + feature flags.
Complete
PHASE 2 — NEXT
GTM Hub
Add email campaigns, social scheduling, content calendar. Still single-tenant but usable for client consulting engagements.
Hard — 4-6 weeks
PHASE 3 — LATER
Multi-Tenant
User accounts, org-level isolation, per-client brand kits and config stores. Each consulting client gets their own workspace.
Complex — 2-3 months
PHASE 4 — FUTURE
SaaS Product
Self-serve onboarding, billing (Stripe), usage tiers, white-label option. Full marketing ops platform with AI copywriting.
Complex — 3-6 months
What Changes Per Phase
Auth: Password → JWT sessions → OAuth/SSO → Stripe-gated
Storage: Single Redis key → Namespaced keys → Per-org isolation → Dedicated instances
Frontend: Vanilla JS → React/Svelte → Component library → White-label theming
AI: Haiku taglines → Campaign copy → Multi-channel generation → Fine-tuned brand models
